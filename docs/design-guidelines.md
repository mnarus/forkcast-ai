## design-guidelines.md

### Emotional tone
Feels like a calm studio in Copenhagen—minimal, warm, and quietly supportive.

---

### Typography
- H1: 32px, semi-bold
- H2: 24px, medium
- H3: 20px, medium
- Body: 16px, regular
- Caption: 14px, light
- Line height: 1.5+
- Font style: clean geometric sans (clarity + calm)

---

### Color system
- Primary: #4F7C82 (muted teal)
- Secondary: #E7EFEF (soft gray-blue)
- Accent: #F2A365 (warm orange)
- Background: #FAFAF9 (off-white)
- Text: #1F2933 (dark slate)

- Contrast: WCAG AA+ compliant

---

### Spacing & layout
- 8pt grid system
- Generous padding (16–24px)
- Mobile-first layout
- Single-column focus
- Clear visual hierarchy

---

### Motion & interaction
- Duration: 150–250ms
- Easing: ease-out
- Interactions:
  - Soft fades
  - Gentle slide-ups
  - Subtle hover/tap feedback

- Example:
  - Completing a meal → soft check animation + fade

---

### Voice & tone
- Calm
- Supportive
- Non-judgmental

**Examples:**
- Onboarding: “Let’s make dinner easier.”
- Success: “You’re all set for the week.”
- Error: “Something went wrong—let’s try again.”

---

### System consistency
- Reuse card-based layouts
- Consistent spacing and typography
- Minimal visual noise
- Inspired by:
  - Apple Human Interface
  - Linear
  - Notion

---

### Accessibility
- Semantic HTML structure
- Keyboard navigation support
- Focus indicators visible
- Contrast ≥ 4.5:1

---

### Emotional audit checklist
- Does it feel calm?
- Does it reduce stress?
- Does it support, not judge?

---

### Technical QA checklist
- Typography follows scale
- Contrast passes WCAG
- Interactive states are clear
- Motion within 150–300ms

---

---

### Design Integrity Review
The system aligns with a calm-first philosophy, minimizing cognitive load while maintaining clarity. One improvement: introduce slightly more warmth in micro-interactions (e.g., subtle delight moments) to deepen emotional connection without breaking calmness.
